# PostgreSQL 전환과 페이지당 문서 JSONB 통저장

Status: ADR-0002를 대체함

서비스급 구조를 목표로 하면서 ADR-0002의 채택 근거였던 "로컬 한 명령 실행" 요구가 폐기되어, SQLite 대신 PostgreSQL(+Drizzle, docker-compose 구동)로 전환한다. 다만 v1은 여전히 싱글유저 워크스페이스(공유·동시 편집 없음)이므로, 문서 본문을 블록 행으로 정규화하지 않고 페이지당 BlockNote JSON 하나를 JSONB 컬럼에 통저장하는 결정은 ADR-0002에서 그대로 계승한다.

## 결과

- 전역 검색은 본문 JSON에서 추출한 텍스트를 저장 시 갱신하고, 한국어는 tsvector 형태소 분석이 빈약하므로 pg_trgm 인덱스 기반 부분 문자열 매칭으로 해결한다.
- 실시간 동시 편집(CRDT)을 나중에 넣으려면 저장 모델 재설계가 필요하다는 제약도 그대로 계승된다.
