# SQLite 단일 파일 DB와 페이지당 문서 JSON 통저장

Status: ADR-0004로 대체됨 (PostgreSQL 전환, JSONB 통저장은 계승)

멀티유저(계정 분리)이지만 동시 편집이 없고, "로컬에서 한 명령으로 실행"이 핵심 요구다. Docker가 필요한 Postgres 대신 설치가 필요 없는 SQLite(+Drizzle)를 쓰고, 문서 본문은 블록을 행 단위로 정규화하지 않고 페이지당 BlockNote JSON 하나로 통저장한다. 블록 정규화의 이점(실시간 동기화, 블록 참조/백링크)이 전부 비목표라 복잡도만 남기 때문이다.

## 결과

- 전역 검색은 본문 JSON에서 추출한 텍스트를 SQLite FTS5 파생 인덱스에 넣어 해결한다(저장 시 갱신).
- 블록 단위 기능(백링크, 블록 동기화, 동시 편집)을 나중에 넣으려면 저장 모델 재설계가 필요하다.
