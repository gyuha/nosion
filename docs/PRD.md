# Nosion PRD — 제품 요구사항 문서 (v1)

노션(Notion)을 클론한 문서 도구. 계정별 독립 워크스페이스에서 블록 기반 문서 편집과 데이터베이스를 제공한다. 이 문서는 v1의 기능 범위와 각 기능의 수용 기준을 정의하며, loop 무인 주행의 기능 목록 원천이다.

- 용어는 [`.forge/CONTEXT.md`](../.forge/CONTEXT.md)의 용어집을 따른다.
- 기술 선택은 [tech-stack.md](./tech-stack.md), 구조는 [architecture.md](./architecture.md), 완료 판정은 [loop-exit-criteria.md](./loop-exit-criteria.md)를 본다.
- 이 문서의 기능 하나는 loop-exit-criteria.md의 E2E 시나리오 하나 이상과 반드시 매핑된다.

## 1. 목표와 사용자

- **목표**: 한 사용자가 자신의 워크스페이스 안에서 페이지를 무한 중첩으로 조직하고, 노션급 UX로 문서를 편집하고, 테이블·보드·리스트 뷰의 데이터베이스를 다룰 수 있는 웹 서비스.
- **사용자**: 이메일·비밀번호로 가입한 개인. 워크스페이스는 사용자 간 공유되지 않는다.
- **v1의 성격**: 백엔드/프론트엔드가 분리된 서비스급 구조(ADR-0003)를 갖추되, 협업 기능은 넣지 않는다.

## 2. v1 기능 범위

### F1. 인증

이메일·비밀번호 기반 가입/로그인/로그아웃. 가입 시 워크스페이스가 자동으로 하나 생성된다.

- 수용 기준:
  - 이메일·비밀번호로 가입하면 로그인 상태가 되고 빈 워크스페이스가 보인다.
  - 로그아웃 후 앱 경로에 접근하면 로그인 화면으로 이동한다.
  - 다른 계정의 페이지는 어떤 API로도 읽거나 쓸 수 없다(워크스페이스 격리).

```
가입 → 워크스페이스 자동 생성 → 로그인 상태 진입
로그인 → 세션 발급(쿠키) → 워크스페이스 진입
로그아웃 → 세션 무효화 → 로그인 화면
```

### F2. 페이지 트리 (사이드바)

페이지는 사이드바 트리에 표시되고 무한히 중첩된다. 문서 페이지와 데이터베이스 페이지 두 종류.

- 수용 기준:
  - 사이드바에서 페이지 생성(문서/데이터베이스 선택), 이름 변경, 삭제(휴지통 이동)가 된다.
  - 페이지 아래에 하위 페이지를 만들 수 있고, 접기/펼치기가 된다. 중첩 깊이 제한 없음.
  - 드래그(또는 이동 메뉴)로 페이지를 다른 페이지 아래·형제 순서로 옮길 수 있다.
  - 트리 상태(순서·계층)는 새로고침 후에도 유지된다.

### F3. 문서 페이지 편집

문서 페이지의 본문은 블록의 나열이며, BlockNote 에디터(ADR-0001)로 편집한다.

- 수용 기준:
  - 문단·헤딩(1~3)·불릿/번호/체크 리스트·코드·인용 블록을 만들고 편집할 수 있다.
  - `/` 슬래시 메뉴로 블록 타입을 선택하고, 드래그 핸들로 블록 순서를 바꿀 수 있다.
  - 텍스트 서식(굵게·기울임·밑줄·취소선·인라인 코드·링크)이 된다.
  - 편집 내용은 자동 저장되며, 새로고침 후 그대로 복원된다.
  - 페이지 제목을 본문 상단에서 편집하면 사이드바 트리에 즉시 반영된다.

### F4. 데이터베이스 페이지

속성 스키마와 행 목록을 본문으로 갖는 페이지. 행을 열면 그 자체가 문서 페이지다.

- **속성 6종**: 텍스트, 숫자, 셀렉트, 다중 셀렉트, 날짜, 체크박스.
- 수용 기준:
  - 속성을 추가·이름 변경·타입 변경·삭제할 수 있다.
  - 행을 추가·삭제하고, 각 속성 타입에 맞는 편집기(텍스트 입력, 숫자 입력, 옵션 선택, 날짜 선택, 체크)로 값을 편집할 수 있다.
  - 행을 열면 문서 페이지가 열리고, 상단에 속성 편집 영역·아래에 본문(블록 편집)이 있다.

### F5. 데이터베이스 뷰 (테이블·보드·리스트)

하나의 데이터베이스는 뷰 여러 개를 가질 수 있고, 각 뷰는 필터와 정렬을 가진다.

- **테이블 뷰**: 속성이 열, 행이 줄인 스프레드시트형 표시. 셀에서 바로 값 편집.
- **보드 뷰**: 셀렉트 속성 하나를 기준으로 행을 열(그룹)로 묶은 칸반. 카드를 드래그해 다른 그룹으로 옮기면 해당 셀렉트 값이 바뀐다.
- **리스트 뷰**: 행을 제목 중심의 세로 목록으로 표시.
- 수용 기준:
  - 뷰를 추가·이름 변경·삭제하고 뷰 간 전환할 수 있다. 기본 뷰는 테이블.
  - 보드 뷰는 그룹 기준 셀렉트 속성을 선택할 수 있고, 셀렉트 속성이 없으면 만들도록 안내한다.
  - 모든 뷰에서 필터(속성 값 조건)와 정렬(속성 기준 오름/내림)이 동작하고, 뷰별로 저장된다.

### F6. 전역 검색

워크스페이스 전체에서 페이지 제목과 문서 본문 텍스트를 검색한다.

- 수용 기준:
  - 검색창(단축키 포함)에 입력하면 제목·본문 매칭 결과가 페이지 목록으로 나온다. 한국어 부분 문자열 매칭이 된다(ADR-0004).
  - 문서 저장 시 검색 인덱스가 갱신된다 — 방금 저장한 본문 텍스트가 즉시 검색된다.
  - 결과를 클릭하면 해당 페이지로 이동한다. 휴지통의 페이지는 결과에서 제외된다.

### F7. 휴지통

소프트 삭제된 페이지의 보관소. 복원하거나 영구 삭제할 수 있다.

- 수용 기준:
  - 페이지 삭제 시 하위 페이지까지 함께 휴지통으로 간다(트리·검색에서 사라짐).
  - 휴지통에서 복원하면 원래 위치로 돌아온다(부모가 휴지통에 있으면 최상위로 복원).
  - 영구 삭제하면 데이터가 실제로 제거되고 복원할 수 없다.

```
페이지 삭제 → 휴지통(소프트 삭제)
                ├─ 복원 → 트리로 복귀
                └─ 영구 삭제 → 데이터 제거(비가역)
```

### F8. 즐겨찾기

페이지를 사이드바 상단 고정 섹션에 꽂아두는 표시. 페이지 트리의 위치는 바뀌지 않는다.

- 수용 기준:
  - 페이지에서 즐겨찾기를 켜고 끌 수 있고, 켜면 사이드바 즐겨찾기 섹션에 나타난다.
  - 즐겨찾기 항목 클릭 시 해당 페이지로 이동한다. 휴지통으로 간 페이지는 섹션에서 사라진다.

### F9. 페이지 아이콘·커버

페이지마다 아이콘(이모지)과 커버(프리셋 색/그라디언트)를 설정할 수 있다. 파일 업로드는 없다.

- 수용 기준:
  - 이모지 피커로 아이콘을 설정·변경·제거할 수 있고, 사이드바 트리와 페이지 상단에 표시된다.
  - 프리셋 목록에서 커버를 설정·변경·제거할 수 있고, 페이지 상단에 표시된다.

### F10. 다크모드

라이트/다크 테마 전환. 선택은 유지된다.

- 수용 기준:
  - 테마 토글로 전환되고, 새로고침·재로그인 후에도 선택이 유지된다.
  - 에디터·데이터베이스 뷰를 포함한 전 화면이 테마를 따른다.

## 3. 비목표 (v1에서 하지 않는 것)

- **워크스페이스 공유·멤버 초대·권한** — 워크스페이스는 1인 소유로 고정.
- **실시간 동시 편집(CRDT)·커서 프레즌스** — 저장 모델 재설계가 필요한 v2 이후 과제(ADR-0004).
- **파일 업로드** — 이미지 블록·첨부·커버 업로드 없음. 아이콘은 이모지, 커버는 프리셋으로 한정.
- **멀티컬럼 레이아웃, PDF/DOCX 내보내기** — BlockNote XL 패키지 라이선스 제약(ADR-0001).
- **백링크·블록 참조·블록 동기화** — 블록 정규화가 전제라 비목표(ADR-0004).
- **공개 페이지 게시, 댓글, 알림, 버전 이력, 오프라인 지원, 모바일 전용 UI.**

## 4. 품질 요구

- 모든 기능은 [loop-exit-criteria.md](./loop-exit-criteria.md)의 E2E 시나리오로 기계 검증 가능해야 한다.
- 워크스페이스 격리는 백엔드에서 강제한다(모든 조회·변경 쿼리에 소유자 조건).
- 문서 자동 저장은 편집 중 데이터 유실이 없어야 한다(저장 실패 시 사용자에게 표시).
